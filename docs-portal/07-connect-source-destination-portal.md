# 07 - Connect Source And Destination - Azure Portal

## Objective

Connect separate source and destination VNets using portal-based peering.

## Steps

1. Open the source VNet.
2. Select Peerings and add a peering to the destination VNet.
3. Allow virtual network access.
4. Open the destination VNet.
5. Add the reverse peering to the source VNet.
6. Confirm both peerings show Connected.
7. Record any NSG, firewall, or DNS requirements.

## Expected Outcome

Approved source-to-destination connectivity exists for transition testing.
