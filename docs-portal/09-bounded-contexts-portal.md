# 09 - Bounded Contexts - Azure Portal

## Objective

Map eShopOnWeb capabilities to destination services.

## Steps

1. Review source code locally while keeping portal open for source resources.
2. Map catalog, basket, ordering, identity, and infrastructure capabilities.
3. Decide what moves to destination first.
4. Record destination resources that will host each capability.
5. Keep source rollback routes visible.

## Expected Outcome

A source-to-destination decomposition map is complete.
