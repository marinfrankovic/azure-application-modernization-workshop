# 13 - Data Decomposition - Azure Portal

## Objective

Plan data ownership in the destination environment.

## Steps

1. Identify source monolith data areas.
2. Create a data ownership matrix.
3. Decide whether destination services need Azure SQL, PostgreSQL, Cosmos DB, or another data store in a future iteration.
4. Store connection references in Key Vault for regulated paths.
5. Document synchronization and rollback.

## Expected Outcome

A data ownership plan exists before production migration.
